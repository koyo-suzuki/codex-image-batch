# Web UI 使い方ガイド

このWeb UIは、画像生成の設定ファイル`jobs.json`と参照画像を準備するためのローカル画面です。UIの「保存」だけでは画像生成は始まりません。保存後にCodexへ実行を依頼します。

## まず覚えること

- 1カード＝1プロンプト＝1画像です
- 100カードあれば、100件分の画像生成呼び出しを待機前に一斉投入します
- 「一斉投入数」は1〜100件です。100件を一気に送りたい場合は`100`を選びます
- 実際の画像生成はCodex内で行います。APIキーは不要です
- 同じ出力画像がすでにあるジョブは、通常はスキップされます

## 1. UIを起動する

### macOS

リポジトリ内の`start-ui.command`をダブルクリックします。ターミナルから起動する場合は次を実行します。

```bash
./start-ui.command
```

ブラウザが自動で開かない場合は、ターミナルに表示された`http://localhost:...`をChromeまたはEdgeで開きます。

### Windows / Linux

```bash
cd ui
npm ci
npm run dev
```

表示されたローカルURLをChromeまたはEdgeで開きます。フォルダへ直接保存する機能はChrome系ブラウザ向けです。

## 2. 制作フォルダを接続する

1. 画面上部の「制作フォルダを接続」を押します
2. `jobs.json`、`references/`、`outputs/`を置きたい制作フォルダを選びます
3. 画面右上が「フォルダ名 に接続中」になったことを確認します

接続したフォルダに`jobs.json`がある場合は、その内容が自動で読み込まれます。ない場合は新しい設定を作成できます。

> フォルダを接続しただけでは保存されません。入力後に必ず「保存して準備完了」を押してください。

## 3. プロンプトを登録する

### 複数のプロンプトをまとめて登録する

1. 「プロンプトをまとめて貼る」に、1行につき1プロンプトを入力します
2. 「行ごとにジョブを作る」を押します
3. 入力した行数と同じ数のジョブカードが作られます

例:

```text
白背景で撮影した高級感のある腕時計の商品写真
夜のネオン街を歩く白猫、映画のワンシーン風
海辺に建つ未来的なホテル、夕暮れ、横長構図
```

この例では3枚を生成します。「行ごとにジョブを作る」を押すと、現在のカードは入力した行の内容に置き換わるため、編集中の内容がある場合は先に保存してください。

プロンプト内で改行を使いたい場合は、一括入力ではなく各カードの「この1画像のプロンプト」で編集します。

### 空の入力欄を100件用意する

「100件の入力欄を用意」を押すと、カードが100件になるまで追加されます。100件では一括入力のほうが速いため、通常は100行のプロンプトを貼り付けて「行ごとにジョブを作る」方法がおすすめです。

### 1件だけ追加する

「＋ 1件追加」を押します。似た設定を作る場合は、元のカードにある「複製」が便利です。複製後はファイルIDとプロンプトを確認してください。

## 4. 各カードを設定する

| 項目 | 使い方 |
|---|---|
| 新規生成 | 参照画像を参考に、新しい画像を作ります |
| 画像編集 | 1枚目の画像を編集対象として扱います |
| ファイルID | ジョブを識別する名前です。ほかのカードと重複させないでください |
| 出力名 | 任意のPNGファイル名です。未入力ならファイルIDから作られます |
| この1画像のプロンプト | 何を、どんな構図・雰囲気で作るかを入力します |
| 縦横比 | `1:1`、`3:2`、`2:3`、`16:9`、`9:16`から選びます |
| 必ず守ること | 変更してほしくない形状、人物、服装、構図などを入力します |
| 入れないもの | 文字、ロゴ、透かしなどの除外要素を入力します |

ファイルIDには、内容が分かる短い英数字を使うと管理しやすくなります。例: `watch-white-01`、`cat-neon-01`。

## 5. 参照画像を追加する

1. 対象カードの「画像をここへドロップ」へPNG・JPEG・WebPをドロップします
2. または枠をクリックして画像を選択します
3. 画像ごとに役割を入力します

役割の例:

- `商品の形状と色を保つ`
- `人物の顔と髪型の参照`
- `背景の色と雰囲気だけ参考にする`
- `編集対象`

「新規生成」では、すべての画像を参照画像として扱います。「画像編集」では、1枚目が編集対象です。2枚目以降は追加の参照画像として使います。

保存時、追加した画像は制作フォルダ内の`references/`へコピーされ、各ジョブ専用のファイル名になります。

## 6. 一斉投入数を決める

画面上部で`10`、`25`、`50`、`100`から選びます。100件を一気に投入する場合は`100`を選びます。

- 25ジョブ・一斉投入数100: 25件を1つのburstで投入
- 100ジョブ・一斉投入数100: 100件を1つのburstで投入
- 150ジョブ・一斉投入数100: 100件、50件の2ウェーブ

各ウェーブでは、全ジョブの呼び出しを先に作成してから完了待ちへ入ります。サービスの混雑や利用枠により、実際の計算開始・完了時刻は前後する場合があります。

## 7. `jobs.json`を保存する

1. 画面下部の「保存して準備完了」を押します
2. Chromeに「変更を保存しますか？」と表示された場合は「変更を保存」を押します
3. 画面下部のメッセージが「jobs.json と参照画像を保存しました」になったことを確認します

保存後の制作フォルダは次のようになります。

```text
制作フォルダ/
├── jobs.json
├── references/
│   ├── image-1-01.jpg
│   └── image-1-02.jpg
└── outputs/        # 生成後に画像と実行記録が入る
```

「画像制作 に接続中」だけでは保存完了ではありません。必ず保存完了メッセージまで確認してください。

## 8. Codexで画像生成を実行する

Codexで同じ制作フォルダを開き、次のように送ります。

```text
$generate-image-batch で jobs.json の各プロンプトを最大100件のburstで一斉開始して
```

または、UI上部の「実行文をコピー」を押してCodexへ貼り付けます。

Codexは次の順で処理します。

1. `jobs.json`と参照画像を確認
2. 出力済みのジョブを判定
3. 設定した一斉投入数の呼び出しを待機前に作成
4. 完成画像を`outputs/`へ保存
5. `run-日時.json`へ実行結果を記録

## 9. 結果を確認する

生成結果は制作フォルダの`outputs/`へ保存されます。

```text
outputs/
├── image-1.png
├── image-2.png
└── run-20260722-025439.json
```

同じ出力名の画像がすでにある場合、そのジョブは自動でスキップされます。作り直す場合はCodexへ次のように依頼します。

```text
$generate-image-batch で jobs.json を、既存画像を上書きして再生成して
```

上書きすると以前の出力を置き換えるため、残したい画像は先に別名へ変更してください。

## よくある問題

### Codexから「jobs.jsonが見つからない」と言われる

- UIで「保存して準備完了」を押したか確認します
- Chromeの保存確認で「変更を保存」を押したか確認します
- UI右上の接続フォルダと、Codexで開いている制作フォルダが同じか確認します
- 制作フォルダ直下に`jobs.json`があるか確認します

### 参照画像が見つからない

- UIから制作フォルダを接続して保存し直します
- `references/`内の画像を保存後に移動・削除していないか確認します
- `jobs.json`だけを別フォルダへ移動しないでください

### フォルダへ直接保存できない

ChromeまたはEdgeを使ってください。未対応ブラウザでは`jobs.json`のダウンロードになります。その場合、参照画像は手動で`references/`へ置き、JSON内のパスと一致させる必要があります。

### 入力したプロンプトが全部同じになった

「複製」はカードの設定をそのままコピーします。複製後に各カードのプロンプト、ファイルID、参照画像を変更してください。

### 100件が完全に同時に終わらない

最大100件の呼び出しは同じburstで投入しますが、画像生成サービス側で順番待ちや速度制限が入る場合があります。完了時刻がずれても、クライアント側の一斉投入が無効とは限りません。実行記録の`dispatched_at_ms`で各ジョブの投入時刻を確認できます。

内蔵画像生成で保証できるのはクライアント側の一斉投入までです。100件の計算処理そのものを同時刻に開始する保証が必要な場合、OpenAI公式ではImage APIによる大量バッチが案内されています。
